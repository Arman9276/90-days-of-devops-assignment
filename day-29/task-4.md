1. Run a Container in Detached Mode

Detached mode means the container runs in the background and does not block your terminal.

Command:

docker run -d nginx

Explanation:

-d → Detached mode

Container runs in background

Terminal is free to use other commands

Example output:

f3c4a1b92a3c9e6a...

This long string is the container ID.

Difference
Mode	Behavior
Interactive (-it)	Runs in terminal
Detached (-d)	Runs in background
2. Give a Container a Custom Name

Instead of using random container IDs, you can assign a name.

Command:

docker run -d --name webserver nginx

Now the container name is webserver.

Check it:

docker ps

Example output:

CONTAINER ID   IMAGE   NAME
abcd1234       nginx   webserver
3. Map a Port from Container to Host

Port mapping allows you to access container services from your computer.

Command:

docker run -d -p 8080:80 --name mynginx nginx

Explanation:

8080 → Host port
80   → Container port

Access it in your browser:

http://localhost:8080

You will see the Nginx welcome page.

4. Check Logs of a Running Container

Logs show the output generated by the container.

Command:

docker logs mynginx

Example output:

172.17.0.1 - - [10/Mar/2026] "GET / HTTP/1.1" 200

This shows requests made to the web server.

To watch logs live:

docker logs -f mynginx

-f means follow logs in real time.

5. Run a Command Inside a Running Container

You can execute commands inside a running container.

Command:

docker exec -it mynginx bash

Explanation:

exec → run command inside container

-it → interactive terminal

bash → open shell

Now you are inside the container.

Example:

ls
cd /usr/share/nginx/html
cat index.html

Exit container:

exit
